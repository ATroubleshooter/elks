Subject: ELKS 0.0.67 on ftp.ecs.soton.ac.uk

ELKS 0.0.67 has been released and is available from the following URL:

ftp://ftp.ecs.soton.ac.uk/pub/elks/elks-0.0.67

As usual three files are available:-

elks-0.0.67.tar.gz - Kernel Source
elkscmd.tar.gz - User space source
images.zip - Boot and root disk images

Major new kernel features in this release:

Firstly the kernel source should now be unpacked in /usr/src/elks rather that
/usr/src/linuc-86/elks. Rob and I have agreed this as elks and dev86 are
independant packages. This will also allow dev86 to use the kernel include
files properlly, and get to the syscall.dat file in the kernel source tree.
This should solve the problem people were having with the wrong version of
syscall.dat being distributed with dev86.

Blaz Antonic's lp driver is now included, with some modifications by me. I don't
know if this works as I have no working printer on my test machines, and no
working printer.

I have added module code to the kernel based on code written by Rob. This allows
drivers to be compiled separatly from the kernel and then loaded into the kernel
at runtime. Only one module can be loaded at once. For information on how
this works see the documentation in elks/Documentation/modules.txt, and see
the files elks/modules/* and kernel/module.c. If there are any qeries I will
answer questions on the list. Included on the root disk is the program
insmod used to install a module, and serial.D and serial.T, the serial
driver compiled as a module. insmod removed the previous module before loading
a new one, so there is no need for rmmod.

The irq code that I was having problems with is now fixed, but the serial
driver still cannot receive characters. I haven't had much time to look at the
code. Output is now much faster thanks to some changes made to both the serial
driver and the tty layer.

Added a CONFIG option which allows the VT52 emulation of the console code to be
removed which frees up a lot of kernel space. Could be useful for those users
who need to squeeze in that extra feature.

Shane Kerr's patch to add time functionality has been added, but is not
tested. Shane, please let me know if it works. If so, is there any chance you
could complete the job and implement the kernel xtime variable correctly
so that file data and time stamps are correctly updated.

Serious debugging code added to the interrupt code so that we can get more
insight into the "panic: unable to get timer" problem.

Many bug fixes, and code trimmings.

User Space

The root disk now contains the fixed version of fdisk, which works fine for me.
The new login program with password support is also included. I tested the
passwd program provided, but it just deleted my passwd file.

Please feel free to send bug reports directly to me. My feed from the list is
still dodgey.

Al
