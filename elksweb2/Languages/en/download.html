CVS (Concurrent Versioning System)

You can also use CVS to get the latest snapshot of the various ELKS parts, either as an ELKS SourceForge Developer (with a username listed on the ELKS SourceForge Project Page), or as a Non-Developer Interested Party.

ELKS SourceForge Developers

This project's SourceForge CVS repository can be checked out by ELKS developers through user CVS with the following instruction set, replacing ${USER} with your assigned SourceForge username:

    CVS_RSH=`which ssh`
CVSROOT=:ext:${USER}@cvs.elks.sf.net:/cvsroot/elks
export CVS_RSH CVSROOT
cvs -d $CVSROOT get modulename

Non-Developer Interested Parties

This project's SourceForge CVS repository can be checked out by non-developers through anonymous pserver CVS with the following instruction set. The module you wish to check out must be specified as the modulename. When prompted for a password for anonymous simply press the <Enter> key.

    CVS_RSH=`which ssh`
CVSROOT=:pserver:anonymous@cvs.elks.sf.net:/cvsroot/elks
export CVS_RSH CVSROOT
cvs -d $CVSROOT login
cvs -d $CVSROOT get modulename

Available Modules

The available modules are:

elks

The ELKS kernel.

elkscmd

Collection of software for ELKS.

elksnet

The ELKS Internet Protocol version 4 netoworking code.

elksutils

Utilities for use with ELKS.

elksweb

The ELKS website, version 1 (obsolete).

elksweb2

The ELKS website, version 2.
